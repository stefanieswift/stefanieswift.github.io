Object-Oriented Programming vs. Functional Program

Which shoud I be using?

February 22

Object-Oriented Programing

Object-oriented programming is based on data structures containing data, called objects. This data is stored in fields, such as attributes, or methods. With object-oriented programming, the data can be changed in the object.

Functional Programming

Functional programming is a declarative programming paradigm that computes data by evaluating mathematical functions. Instead of storing data in attributes and methods, in functional programming the data is stored in expressions and arguments.

What should I be using?

In ruby, the two paradigms are not opposite languages. They should be used together to write efficient code. When you are writing a new program in ruby, it is best to use object-oriented programming for the code overall and to implement functional programming inside individual methods.
