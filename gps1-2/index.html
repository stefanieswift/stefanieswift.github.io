Pair Programming

  • Pair programming is when two programmers work together at the same time.
  • The programmers switch roles to garner communication.
  • Often, one programmer is a driver and one is a navigator.

Why is Pair Programming Important

  • Pair programming is important because it is an opportunity for two programmers to work together.
  • It allows programmers an understanding of how people work.
  • Programmers can learn new skills and share each other's knowledge to work through systems that one might not be familiar with.

Driver vs Navigator

  • Entering a pairing session, the two parties have two roles to make work more efficient.
  • The navigator instructs the driver on what to program.
  • The driver does the actual coding based on the navigator's instructions.
